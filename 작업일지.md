# 작업일지

## 2026-04-19 — 카톡 임포트 · 레포 기반 개발 · 홈 리디자인

### 배경
- 사용자가 제기한 4가지 이슈:
  1. 카카오톡 `.txt` 파일을 올려도 각 링크가 제대로 열리지 않고, 기능·해시태그·제목이 제대로 분석되지 않음.
  2. GitHub 레포를 기반으로 추가 개발(develop)을 할 수 있는 플로우가 필요.
  3. 그 외에 유용한 기능을 추가로 디벨롭.
  4. `https://stashby.me/` 와 홈 인터페이스가 비슷해 표절 의혹 방지를 위한 UI 차별화 필요.

---

### 1. 카카오톡 링크 스캔 고도화
**파일:** `components/Sidebar.tsx`, `lib/classifier.ts`, `lib/batch.ts`(신규)

- URL 파싱 정규식을 `/https?:\/\/[^\s<>"')\]]+/g` 로 타이트하게 고치고, 꼬리에 붙는 `,.!?)` 등 구두점을 정리.
- **모든 링크**에 대해 메타데이터를 수집하도록 기존 `if (index < 10)` 제한을 삭제.
- `lib/batch.ts` 의 `mapWithConcurrency(items, limit, worker, onProgress)` 로 동시성 6 배치 처리.
- 사이드바 하단에 진행률 바(`N/Total`) 추가 — 대량 임포트 중에도 상태가 보이게 됨.
- `lib/classifier.ts` 를 전면 재작성:
  - 도메인 규칙을 30+ 케이스로 확장 (개발/디자인/영상/지식블로그/생산성/AI/쇼핑/뉴스/재테크).
  - 경로(path) 기반 보조 분류 (`/docs`, `/blog`, `/product` 등).
  - 메타 제목·설명을 2차 신호로 받아 `#React`, `#TypeScript`, `#Python`, `#DevOps`, `#AI` 같은 기술 해시태그 자동 부여.
  - `buildAutoTitle`, `buildAutoDescription` 헬퍼로 메타 없을 때 도메인·마지막 path 세그먼트 기반 제목/설명 생성.
- `AddLinkModal` 도 같은 헬퍼를 써서 수동 추가 시에도 일관된 품질.

**효과:** 100개짜리 카톡 대화록도 모든 링크가 실제 OG 메타로 채워지고, 해시태그·카테고리가 신뢰할 만한 수준으로 나옴.

### 2. "Develop from Repo" — 레포 기반 아이디에이션
**파일:** `lib/github.ts`, `lib/repoIdeas.ts`(신규), `components/RepoDevelopModal.tsx`(신규), `contexts/AppContext.tsx`, `components/Sidebar.tsx`, `app/page.tsx`

- 사이드바의 각 레포 항목에 `GitBranch`  **Develop** 버튼 추가.
- 모달이 열리면 GitHub에서 세 가지를 동시 fetch:
  - 루트 디렉터리 트리 (`contents`)
  - README 원문 (`readme`)
  - 언어 통계 (`languages`)
- `lib/repoIdeas.ts` 의 `generateIdeas()` 가 Signals(테스트/CI/Dockerfile/Next.js/Firebase 등) 유무를 보고 구현 프롬프트 3~6개 자동 생성.
  - 예) 테스트 설정이 없으면 → "테스트 스위트 도입" 프롬프트 + 운영 워크플로 템플릿
  - Next.js + React 감지 시 → 에러 바운더리/Sentry/헬스엔드포인트 도입
  - Firebase/Supabase 감지 시 → 규칙·RLS 리뷰
  - Python 감지 시 → pyproject 현대화
- 각 아이디어 카드의 `프롬프트 복사` 버튼을 누르면 곧바로 Claude/다른 AI 에이전트에 붙여넣을 수 있는 상세 지시문이 클립보드에 들어감.

**효과:** 내가 올린 레포 소스를 바탕으로 "다음에 뭘 만들지" 를 곧바로 AI 한 턴만에 실행 가능한 형태로 얻을 수 있음.

### 3. 추가 기능 — 선택 모드 · 태그 필터 · 시간 그룹
**파일:** `contexts/AppContext.tsx`, `components/LinkCard.tsx`, `app/page.tsx`

- **벌크 선택 모드:** 상단 `선택` 토글 → 카드 좌상단 체크박스 노출. 하단에 플로팅 툴바 `N개 선택됨 · 삭제 · 취소`.
- **`removeManyLinks`** 액션으로 Firestore에서 선택된 문서 일괄 삭제.
- **태그 칩바:** 전체 링크에서 상위 14개 태그를 HOT 태그로 노출, 클릭하면 `activeTag` 로 필터. 카드 내 `#태그` 도 클릭으로 같은 필터링 가능.
- **시간 그룹:** 피드를 `오늘 / 지난 7일 / 이전` 으로 자동 구간화하여 시각적 레이어 생성.

### 배경 / 피드백

- 사용자 요청: **GitHub 로그인하면 내 레포를 자동 분석해서, 레포마다 "지금 이거 해보세요" 카드를 깔끔하게 보여주는 추천 대시보드로 바꾸자.**
- 추가로 우상단의 원형 이니셜 로그아웃 버튼이 어색 → 계정 pill + 드롭다운으로 정리.

### 1. 홈 히어로 아래에 `내 레포에 지금 필요한 것` 섹션 추가
**파일:** `components/RepoInsights.tsx`(신규), `app/page.tsx`

- GitHub 토큰이 있을 때, 최근 업데이트된 레포 상위 6개를 동시에 분석 (`mapWithConcurrency` 동시성 3).
- 각 레포마다 `fetchRepoContents + fetchRepoReadme + fetchRepoLanguages` → `generateIdeas()` 로 Top 1 개선 포인트 추출.
- 카드 구조
  - 좌: 아이콘(🧪 테스트 · 📘 README · 🔒 보안 · 📊 관찰 · 📦 패키징 · ⚙️ CI · 🔑 env · ✨ 기본)
  - 상단: 레포명 + 주 언어
  - 본문: 개선 제목 + 한줄 이유(2줄 클램프)
  - 액션: `자세히 보기` (Develop 모달 열기) + 아이콘 버튼 `프롬프트 복사`
- 로딩 중엔 스켈레톤 카드 3개 표시.

**효과:** 프롬프트 덩어리를 먼저 들이밀지 않고, 한눈에 "어떤 레포에 뭐가 비어있는지"만 보여줌. 상세는 필요할 때만 모달로.
